# Weekly Activation Report Style Contract

Last updated: 2026-02-20

Purpose:
- Keep weekly output decision-oriented under low-volume conditions.
- Force evidence-first writing with explicit confidence cues.

Contract:
1. Executive Summary
- One factual sentence.
- Must reflect North Star direction and include risk/context cues.
- Avoid repeating exact metric values that are already shown in Key Metrics.

2. Key Changes
- 2-3 highest-priority movements.
- Every item includes:
  - chart title + link
  - percentage change or conversion rate
  - absolute counts (for example `42/100` or absolute deltas)

3. Insights & Next Steps
- Grounded only in provided chart evidence, feedback themes/snippets, release context, and memory.
- Pair each insight with a concrete next step in the same bullet.
- If evidence is thin, claim text must include `Low confidence:`.
- Avoid speculative language that is not connected to supplied evidence.

4. Action Formatting
- Immediate and concrete.
- Include ownership and the next verification/investigation step.
- Avoid explicit priority labels unless specifically requested.
- Avoid filler labels such as `Expected impact:` when the action is already clear.

5. Reliability Cues
- Use chart reliability annotations to adjust confidence wording:
  - low sample size -> directional read only
  - incomplete latest bucket -> caution on period-over-period interpretation

6. PII Safety
- No raw emails or direct user identifiers in final report text.
